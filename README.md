Each platform has its own I/O primitive вҖ” epoll on Linux, kqueue on macOS, Netty on JVM.
keel unifies them behind a single Kotlin Multiplatform interface, giving you the right foundation for implementing network protocols across all targets.

- **Native-first**: drives epoll (Linux) and kqueue (macOS) directly from Kotlin Native
- **Suspend API**: all I/O operations are `suspend fun`, ready for async event loop
- **Platform-optimal**: epoll/kqueue on Native В· NIO/Netty on JVM В· net module on Node.js
- **Codec layer**: HTTP/1.1 and WebSocket built on pure `kotlinx.io` primitives

## Roadmap

### 0.3.0 (current вҖ” experimental)

- 6 I/O engines: epoll, kqueue, NIO, Netty, NWConnection, Node.js
- SlabAllocator (Native) + PooledDirectAllocator (JVM) for per-EventLoop buffer pooling
- EventLoop dispatch for Native engines
- Deferred flush + writev batching
- Logging module (zero-dependency Logger interface)
- Exception hierarchy (KeelEofException, HttpParseException)
- detekt static analysis + TrackingAllocator leak detection

### Next (Phase 7)

- TLS (Mbed TLS)
- Adaptive buffer + Write water mark
- iOS targets

### Future (Phase 8+)

- io_uring engine (Linux 5.1+)
- Push Channel + keel native server (:server, Ktor-independent)
- UDP transport
- HTTP/2, gRPC, MQTT
- HTTP/3 (QUIC)

## Installation

> **Note:** keel is not yet published to Maven Central. Build from source and publish to your local Maven repository.

```bash
git clone https://github.com/fukusaka/keel.git
cd keel
./gradlew publishToMavenLocal
```

Then add the dependency to your project:

```kotlin
// build.gradle.kts
repositories {
    mavenLocal()
}

dependencies {
    // Ktor + keel server engine
    implementation("io.github.fukusaka.keel:keel-ktor-engine:0.3.0")
    implementation("io.ktor:ktor-server-core:3.4.1")

    // Low-level I/O (without Ktor)
    implementation("io.github.fukusaka.keel:keel-core:0.3.0")

    // Codecs (optional)
    implementation("io.github.fukusaka.keel:keel-codec-http:0.3.0")
    implementation("io.github.fukusaka.keel:keel-codec-websocket:0.3.0")
}
```

Maven Central publication is planned for a future release.

### Notes

- All keel engines use fully async I/O with HTTP/1.1 keep-alive.
- **native:ktor-keel-epoll** (589K) outperforms **native:ktor-cio** by 65x on Linux, reaching 45% of Rust.
- **native:ktor-keel-kqueue** (108K) is 70% of Rust on macOS. On /large (100KB), kqueue reaches 95% of Rust (57K vs 60K).
- **jvm:ktor-keel-nio** (128K on macOS, 540K on Linux) approaches **jvm:ktor-netty** performance.
- Phase 6b improvements: EventLoop dispatch (+27-39% /hello), deferred flush + writev batching (+210-578% /large), SlabAllocator/PooledDirectAllocator for per-EventLoop buffer pooling.
